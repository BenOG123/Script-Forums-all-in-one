C# Commands for Unity

Essential C# commands and functions for game development in Unity

Movement and Transform Commands

transform.Translate()
Moves the transform in the direction and distance of translation.
transform.Translate(Vector3.forward * speed * Time.deltaTime);

How to use:

  1. Use this in the Update() method for continuous movement.
  2. Multiply by Time.deltaTime to make movement frame-rate independent.
  3. Can use Vector3.forward, Vector3.right, etc., or create custom vectors.
  4. Combine with Input.GetAxis() for player-controlled movement.
transform.Rotate()
Rotates the transform by the specified angles.
transform.Rotate(0, rotationSpeed * Time.deltaTime, 0);

How to use:

  1. Use in Update() for continuous rotation.
  2. The parameters represent rotation around x, y, and z axes respectively.
  3. Multiply by Time.deltaTime for smooth, frame-rate independent rotation.
  4. Can be used with Input.GetAxis() for player-controlled rotation.
transform.LookAt()
Rotates the transform so the forward vector points at target's current position.
transform.LookAt(targetTransform);

How to use:

  1. Useful for making an object face another object (e.g., enemy facing player).
  2. Can be used in Update() for continuous tracking.
  3. The target can be any Transform in your scene.
  4. Can be combined with Vector3.Lerp() for smoother rotation.

Input Commands

Input.GetAxis()
Returns the value of the virtual axis identified by axisName.
float horizontalInput = Input.GetAxis("Horizontal"); float verticalInput = Input.GetAxis("Vertical");

How to use:

  1. Use in Update() to continuously check for input.
  2. "Horizontal" axis is usually mapped to A/D or left/right arrow keys.
  3. "Vertical" axis is usually mapped to W/S or up/down arrow keys.
  4. Returns a float between -1 and 1, useful for smooth movement.
  5. Can be used with transform.Translate() for player movement.
Input.GetKeyDown()
Returns true during the frame the user starts pressing the key identified by name.
if (Input.GetKeyDown(KeyCode.Space)) { Jump(); }

How to use:

  1. Use in Update() to check for key presses.
  2. Only returns true on the initial press, not while holding the key.
  3. Useful for actions that should happen once per key press (e.g., jumping, shooting).
  4. Can use KeyCode enum for specific keys, or string names like "space".
Input.GetMouseButtonDown()
Returns true during the frame the user pressed the given mouse button.
if (Input.GetMouseButtonDown(0)) { Shoot(); }

How to use:

  1. Use in Update() to check for mouse clicks.
  2. 0 is left click, 1 is right click, 2 is middle click.
  3. Only returns true on the initial click, not while holding the button.
  4. Useful for actions like shooting, selecting objects, etc.

Physics Commands

Rigidbody.AddForce()
Adds a force to the Rigidbody.
rigidbody.AddForce(Vector3.up * jumpForce, ForceMode.Impulse);

How to use:

  1. Requires a Rigidbody component on the GameObject.
  2. Use ForceMode.Impulse for instant force application (good for jumps).
  3. Use ForceMode.Force for continuous force application.
  4. Can be used in FixedUpdate() for physics-based movement.
  5. Combine with Input commands for player-controlled physics movement.
Physics.Raycast()
Casts a ray against all colliders in the scene and returns detailed information on what was hit.
RaycastHit hit; if (Physics.Raycast(transform.position, transform.forward, out hit, 100)) { Debug.Log("Hit " + hit.collider.name); }

How to use:

  1. Useful for line of sight checks, shooting mechanics, object detection.
  2. Can be used in Update() for continuous checking.
  3. The out parameter 'hit' contains information about what was hit.
  4. Can be combined with Debug.DrawRay() to visualize the ray in the Scene view.
  5. Use LayerMask to selectively ignore certain objects.

Game Object and Component Commands

GameObject.FindGameObjectWithTag()
Returns one active GameObject tagged tag. Returns null if no GameObject was found.
GameObject player = GameObject.FindGameObjectWithTag("Player");

How to use:

  1. Useful for finding specific objects in the scene.
  2. Best used in Start() or Awake(), as it's relatively slow for frequent use.
  3. Make sure the target object has the correct tag assigned in the Unity Inspector.
  4. Returns null if no object with the tag is found, so check for null before using.
GetComponent<T>()
Returns the component of Type T if the game object has one attached, null if it doesn't.
Rigidbody rb = GetComponent<Rigidbody>(); if (rb != null) { rb.AddForce(Vector3.up * 10); }

How to use:

  1. Use to access other components on the same GameObject.
  2. Best used in Start() or Awake() and stored in a variable for later use.
  3. Can be used with any component type (e.g., Collider, Renderer, custom scripts).
  4. Always check for null before using the returned component.

Coroutines

StartCoroutine()
Starts a coroutine, which can be used for actions that take place over time.
StartCoroutine(WaitAndPrint(5f)); IEnumerator WaitAndPrint(float waitTime) { yield return new WaitForSeconds(waitTime); Debug.Log("Printed after " + waitTime + " seconds"); }

How to use:

  1. Useful for actions that need to happen over time or with delays
